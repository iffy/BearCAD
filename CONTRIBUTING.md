# Contributing to BearCAD

BearCAD is an experiment to see what AI can do: the application — geometry kernel
integration, tools, documentation, screenshots, this file — is built almost entirely by an
AI working from plain-English feature requests and bug reports.

Keeping that experiment clean means keeping the authorship rule simple:

## Code contributions: AI only

**Human-written pull requests are not accepted.** All code, documentation, and asset
changes are produced by the AI from issues and review feedback. If you'd like something
built or fixed, describe it — don't build it.

## Humans: please file issues!

Bug reports and feature requests from humans are very welcome, and they're the fuel the
whole experiment runs on:

- **Bugs** — say what you did, what you expected, and what happened instead. Screenshots
  help. If you can reproduce it with a Lua script (Help → Export Session Commands…),
  attach that.
- **Feature requests** — describe the outcome you want in plain English, the way you'd
  explain it to a colleague. You don't need to propose an implementation.

File them at <https://github.com/iffy/BearCAD/issues>.
